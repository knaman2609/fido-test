# Document Editor

A modern, block-based document editor built with BlockNote and TypeScript. Create rich documents with support for text, headings, lists, and more, with automatic localStorage persistence.

## Features

- **Block-based editing**: Notion-like editing experience with BlockNote
- **Rich text support**: Paragraphs, headings, bullet lists, numbered lists, and more
- **Auto-save**: Documents are automatically saved to localStorage
- **Persistent storage**: Your documents persist between browser sessions
- **Clean, responsive UI**: Modern interface with Mantine styling
- **Error handling**: Graceful handling of storage errors and quota exceeded

## Quick Start

```bash
# Install dependencies
npm install

# Build and run
npm start
```

Then open your browser to `http://localhost:3000`

## Prerequisites

- [Node.js](https://nodejs.org/) (version 14 or higher)
- npm (comes with Node.js)

## Installation

1. Clone or download this repository
2. Navigate to the project directory
3. Install dependencies:

```bash
npm install
```

## Running the App

### Option 1: Build and Serve (Recommended)

```bash
npm start
```

This compiles TypeScript and starts a local server at `http://localhost:3000`.

### Option 2: Build Only

```bash
npm run build
```

Then open `index.html` directly in your browser.

### Option 3: Development Mode (Watch)

```bash
npm run watch
```

In another terminal:
```bash
npm run serve
```

The `watch` command automatically recompiles TypeScript when you make changes.

## How It Works

1. **BlockNote Editor**: Uses `@blocknote/core` and `@blocknote/mantine` for a rich block-based editing experience
2. **React Integration**: BlockNote components are rendered using React and ReactDOM
3. **TypeScript Compilation**: Source files in `src/` are compiled to JavaScript in `dist/` using the TypeScript compiler (`tsc`)
4. **Module System**: Uses ES modules (type: "module") for clean imports/exports
5. **Local Storage**: Documents are persisted in the browser's localStorage under the key `blocknote-document`
6. **Auto-save**: Changes are automatically saved on every editor update

## Document Structure

Documents are stored as JSON with the following structure:

```typescript
interface Document {
  id: string;           // Unique document identifier
  title: string;        // Document title
  blocks: Block[];      // Array of BlockNote blocks
  createdAt: number;    // Creation timestamp
  updatedAt: number;    // Last update timestamp
}
```

## Troubleshooting

### "Cannot find module" errors
Make sure you've run `npm install` to install all dependencies including BlockNote and React.

### "dist/ folder not found"
Run `npm run build` to compile TypeScript files.

### Changes not reflecting
- Ensure you're running `npm run watch` for auto-compilation
- Hard refresh your browser (Ctrl+F5 or Cmd+Shift+R)
- Check browser console for errors

### Storage quota exceeded
If you see a warning about storage being full, try clearing your browser's localStorage or use a different browser profile.
